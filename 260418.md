# 260418 세션 이어하기 메모

## 1) 오늘까지 확정된 핵심
- `omx_motion_server` 이중 노드 이슈의 근본 원인은 `launch`에서 `name='motion_server'` 지정으로 들어간 글로벌 remap(`__node:=motion_server`)이 MoveIt 내부 private node에도 적용된 것.
- 조치: `src/omx_motion_server/launch/motion_server.launch.py`에서 `name='motion_server'` 제거.
- `motion_server.cpp`는 helper node(`motion_server_moveit`) 분리 + helper 노드에 파라미터 오버라이드 전달 로직 반영됨.
- perception 결과는 `default_cam` 프레임 기준이며, `world` 변환은 TF 체인(`link5 -> camera_mount -> default_cam`)으로 처리.
- 3cm 큐브 기준 grasp 높이 보정은 `grasp_z_offset` 사용.

## 2) 내일 시작 목표
1. 액션 서버 등록 상태 정상 확인 (`/omx/move_to_pose` 등 `Action servers: 1`).

3. pre-grasp 단일 이동 테스트 성공.

## 5) 파라미터 세팅 (3cm 큐브 기준)

`pick_place_skill` 실행 후:
```bash
ros2 param set /pick_place_skill approach_orientation.x 0.002
ros2 param set /pick_place_skill approach_orientation.y 0.285
ros2 param set /pick_place_skill approach_orientation.z -0.003
ros2 param set /pick_place_skill approach_orientation.w 0.958

ros2 param set /pick_place_skill grasp_z_offset 0.0062
ros2 param set /pick_place_skill pre_grasp_z_offset 0.08
ros2 param set /pick_place_skill lift_z_offset 0.12
ros2 param set /pick_place_skill velocity_scale 0.12
```

노트:
- `grasp_z_offset`은 최근 측정값(`detected bz ~= 0.0088`) 기준으로 큐브 중심 높이(`0.015`)에 맞춘 값.
- 내일 첫 프레임에서 `bz`가 달라지면 `grasp_z_offset = 0.015 - bz`로 재계산.

## 6) 실행 순서 (안전)

1. pre-grasp 단일 이동 먼저:
```bash

"{target_pose: {header: {frame_id: world}, pose: {position: {x: 0.2281, y: 0.0471, z: 0.0950}, orientation: {x: 0.002, y: 0.285, z: -0.003, w: 0.958}}}, velocity_scale: 0.12}"
```

2. 이상 없으면 full pick/place:
```bash
ros2 action send_goal /omx/pick_place omx_interfaces/action/PickPlace \
"{object_color: red, target_box: left, retry_on_fail: false}"
```

## 7) 문제 발생 시 즉시 확인
- `Waiting for an action server...`:
```bash
ros2 action info /omx/move_to_pose
```
`Action servers: 1` 아니면 `motion_server`/도메인/런치 상태부터 재점검.

- TF 변환 실패:
```bash
ros2 run tf2_ros tf2_echo world default_cam
```
결과 없으면 T5/T6 static TF 프로세스 확인.

- 플래닝 실패(`GOAL_STATE_INVALID`):
  - orientation 우선 재확인
  - 그 다음 목표 z 및 workspace 범위 확인
