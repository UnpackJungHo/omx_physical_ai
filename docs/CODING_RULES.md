# OMX 코드 작성 규칙

## 인터페이스 규약
- 패키지 간 통신은 반드시 `omx_interfaces`에 정의된 타입을 사용한다.
- 임시 문자열, ad-hoc 딕셔너리, 비공식 토픽 계약을 만들지 않는다.

## 스킬과 복구 규약
- 스킬 함수는 성공 또는 실패를 명확히 반환한다.
- 실패 처리와 재시도 정책은 `omx_recovery_manager`가 담당한다.
- 스킬 내부에 복구 정책을 하드코딩하지 않는다.

## 모션 규약
- workspace 제한은 MoveIt2 Planning Scene 기준으로 유지한다.
- 안전 제약을 우회하는 직접 제어 코드를 넣지 않는다.
- 저수준 제어 책임은 `omx_motion_server` 바깥으로 새지 않게 유지한다.

## 구현 규약
- 아키텍처 경계를 넘는 편의 구현보다 책임 분리를 우선한다.
- 상위 계층은 하위 계층의 내부 구현을 직접 참조하지 않는다.
- 변경이 구조를 흔들면 코드보다 문서 계약을 먼저 수정한다.
